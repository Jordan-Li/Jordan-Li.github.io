从零开始用Hexo搭建一个博客

 2020-02-16  2020-02-23
  Hexo
  11 minutes read (About 1671 words) 0 visits

前言

​ 这一切的一切都要从一只蝙蝠说起，疫情爆发以后每天宅在家里面，为了不让自己闲得发霉，决定将之前注册的域名利用起来，搭建一个自己的博客。在搭建的过程中碰到了不少问题，但都逐步解决，于是有了下面这篇文章，可以给以后还需要搭建hexo的小朋友一点参考，也给自己一些参考（万一某天我又要搭多一个并且忘了之前怎么搭的了呢？）

前期配置

下载 Nodejs

​ Nodejs的下载可以直接到Nodejs官网去下载，安装左手边LTS版本即可。

下载 Git 并配置

​ 因为在安装hexo的过程中需要利用到Git去抓包，因此Git的安装和配置是不可以少的。在b站上有关于配置Git和汉化的教程，《如何在8分钟内完成Git配置》，按照视频中操作安装即可。笔者在最后一步测试能否顺利抓取Github上的数据的时候遇到了报错，经过查询资料发现是之前在电脑中保存的Github的账号密码出现了问题，该账密是永久储存在电脑中的，如果有变动的话需要重新设置，设置方法。经过重新调整后，已经能够用Git顺利抓取数据了。

Git 抓取速度的提升

​ 在使用Git的时候，笔者发现抓取数据的速度宛如龟速，实在有点难顶。经过高人指教，可以利用代理提升Git的抓取速度，方法在此。经过调教后，Git的抓取速度成功从10KB/s飙升到400KB/s，有点香。

初始化配置Hexo

新建一个文件夹

使用 hexo 初始化博客

这个过程需要一点时间，hexo 会帮我们自动初始化所有的内容并且下载预设主题。

启动博客

hexo 就会在本地的4000端口（localhost:4000) 为你创建你的博客，这个时候 hexo博客已经创建完成了。

在Hexo上创建博文

​ 即可在本地创建一篇新的文章。在本地source文件夹的子文件夹中，文件形式为md文件。用Typora或者其他markdown编辑器打开后即可开始编辑。

​ 编辑完成后，通过hexo g即可生成对应的html文件，再用hexo s即可把博文显示在本地的博客上。（ hexo s可以作为本地测试使用，提前预览博客）

浏览自己的博客

​ 经过上述的步骤后，博客已经储存在GitHub上了，这个时候只要打开 https://***.github.io 即可看到自己的博客辣。

将Github仓库与自己的域名连接

​ 前面有提到，我搭建博客的目的是为了把我之前闲置的域名用起来，那当然少不了把GitHub仓库的GitHub Page和我域名连接起来的过程辣，在阿里云注册好自己的域名以后，就可以开始操作了。

把解析权从阿里云转到Cloudflare

​ 到Cloudflare官网注册后，根据指引点击 Add Site，添加自定以的域名。扫面完成后，Cloudflare会选择给我们分配两个NS地址，登录阿里云（你注册域名的地方），把这两个地址替换掉原来的NS地址，等待生效即可。

修改DNS（解析记录）

​ 为了实现将Github Page和自己的域名连接，就需要把域名解析到Github Page上，添加CNAME和A解析记录，使得www网址和@(即本身) 都指向自己的Github Page。按照以下表格分别添加A和CNAME两条解析记录。

类型Type 主机Host 指向
A @ Github Page 的IP地址
CNAME www Github Page 的网址

其中Github Page的IP地址可以在站长之家ping一下查到，然后填写进去，Github Page的地址就是 ***.git.io。
